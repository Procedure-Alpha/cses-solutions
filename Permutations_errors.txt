1. Base case (n = 1) skip karna

Bahut log directly “NO SOLUTION” print kar dete hain jab n ≤ 3 likh dete hain.
Par actually n = 1 valid hai, kyunki single element me koi adjacent difference hota hi nahi.

Wrong:

if (n <= 3) cout << "NO SOLUTION";


Correct:

if (n == 1) cout << 1;
else if (n == 2 || n == 3) cout << "NO SOLUTION";



2. Condition for NO SOLUTION galat likhna

Kuch log n < 4 likh dete hain, ya sirf n == 2 check karte hain.
Actual mathematical proof se only n = 2 and n = 3 impossible hain.

Example:

n=2 → only permutations [1,2] or [2,1] → both differ by 1 → invalid
n=3 → all permutations fail adjacency rule


So correct condition:
if (n == 2 || n == 3)



3. Wrong print order (most important)

Yeh sabse common trap hai.
Agar tum odd numbers pehle print karte ho, adjacency difference 1 aa jata hai beech me.
Hamesha even pehle print karo, phir odd.

Wrong Output Example (odds first):

Input: 4
Output: 1 3 2 4
|3-2| = 1 → invalid


Correct Output (evens first):

Input: 4
Output: 2 4 1 3
|4-1| = 3 → valid


Explanation:
Evens sequence (2,4,6,...) aur odds sequence (1,3,5,...) ke beech transition gap hamesha ≥ 2 hota hai,
isliye koi pair differ by 1 nahi hota.
